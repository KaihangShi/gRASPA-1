Simulation BoxesÂ¶

  • Here are some of the features related to the simulation boxes for cuda-RASPA

  • NOTE:

    1. These inputs determines simulation mode

    2. These inputs can be used to simulate NVT-Gibbs in two/more boxes

    3. One can simulate the boxes at the same time, or sequentially

    4. To do NVT-Gibbs, use Empty-Box.cif for FrameworkName

    5. The code checks whether the simulation boxes have zero framework atoms in both boxes

      • if so, then it runs NVT-Gibbs (if NVT-Gibbs moves are also used)

      • if not, then the simulation is reduced to running NVT/GCMC simulations in both boxes independently in parallel

  • Input Command NumberOfSimulations:

    • (integer) Number of simulation boxes

    • Usage:

      1. Sets Number of simulation boxes

  • Input Command SingleSimulation:

    • (boolean) yes/no

    • Usage:

      1. whether to simulate Only one simulation box (default simulation box zero)

      2. If running NVT-Gibbs, set NumberOfSimulations 2 and SingleSimulation no

      3. Setting SingleSimulation no, it will use function Run_Simulation_MultipleBoxes in axpy.cu

      4. Setting SingleSimulation yes, it will use function Run_Simulation_ForOneBox in axpy.cu

  • Input Command FrameworkName:

    • list of (integer) Name of the framework file(s)

    • Usage:

      1. Length of this input command is dependent on the input of NumberOfSimulations

      2. Make sure these commands are consistent
