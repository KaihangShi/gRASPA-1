  1. SingleMove

    • inputs:

  2. get_new_position (CUDA FUNCTION)

    • inputs:

    Atoms* d_a, Atoms Mol, Atoms NewMol, ForceField FF, size_t start_position, size_t SelectedComponent, double* random, size_t offset, double3 Max, bool* device_flag, int MoveType

    • Descriptions:

      Process the new and old positions for the selected molecule or group of atoms

    • NOTE:

      1. For the new positions, the xyz locations are determined by the MoveType

      2. For translation, it determines the new xyz based on the maximum displacement in each direction

      3. For rotation, it uses the rotation matrix to rotate the molecule by x-, y-, and z-axis based on the maximum rotation angle

      4. variable device_flag is zeroed at this step, it is used to check for energy overlap in the following energy calculation step

        It is useful to reject the move and skip the expensive Ewald calculation if the newly proposed position is bad.

  3. RotationAroundXAxis/RotationAroundYAxis/RotationAroundZAxis

    • inputs:

    Atoms Mol,size_t i,double theta

    • Descriptions:

      Rotate the group of atoms by theta degrees

  4. update_translation_position (CUDA FUNCTION)

    • inputs:

    Atoms* d_a, Atoms NewMol, size_t start_position, size_t SelectedComponent

    • Descriptions:

      write the new positions (and related values) from NewMol to the start_position of d_a

    • NOTE:

      The start_position is where the molecule should be stored in d_a for the SelectedComponent. For example, for TIP4P (4 atoms), the molecule we selected is 4, then start_position = 4*4 = 16. If SelectedComponent = 1, then we need to update position 16, 17, 18, and 19 in d_a[1]
