  1. CreateMolecule_InOneBox

    • Inputs:

    Components& SystemComponents, Simulations& Sims, ForceField FF, RandomNumber Random, WidomStruct Widom, std::vector<size_t>& NumberOfCreateMolecules, bool AlreadyHasFractionalMolecule

    • Description:

      Do Insertion moves to insert molecules until the required number of molecules are created

    • NOTES:

    1. This function also creates fractional molecules in the box, if it reads a restart file and the restart file has fractional molecule, then no need to create fractional molecule

    2. The create molecule function uses the same algorithm as a normal Insertion move. However, the acceptance rate is modified so that the move can be accepted even when the Rosenbluth weight is super low.

    3. There is a limit on number of failure trials to create molecules. This is for preventing an infinite while loop if the required number is too big.

  2. Run_Simulation_MultipleBoxes

  • Inputs:

    int Cycles, std::vector<Components>& SystemComponents, Simulations*& Sims, ForceField FF, RandomNumber Random, std::vector<WidomStruct>& Widom, std::vector<SystemEnergies>& Energy, int SimulationMode

    • Description

      Run Simulations for multiple boxes for a number of cycles

    • NOTES:

    1. The simulation is classified in different modes: INITIALIZATION, EQUILIBRATION, PRODUCTION

    2. The code first loops over the cycles, then for each cycle, loop over the simulation boxes

    3. FOR FUTURE: multiple boxes can be run in parallel, using cuda streams + openmp
